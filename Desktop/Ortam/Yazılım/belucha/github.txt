# Belucha - Git Komutları

## İlk Kurulum

### Repository'yi Clone Et
```bash
git clone https://github.com/murathan-cotuk/belucha.git
cd belucha
```

## Main Branch İşlemleri

### Main Branch'e Geç
```bash
git checkout main
# veya
git checkout -b main
```

### Main Branch'ten Pull (Güncelleme Çek)
```bash
git pull origin main
```

### Main Branch'e Push (Değişiklikleri Gönder)
```bash
# Değişiklikleri ekle
git add .

# Commit yap
git commit -m "Commit mesajı buraya"

# Main branch'e push
git push origin main
```

### Main Branch'te Değişiklikleri Kontrol Et
```bash
# Durum kontrolü
git status

# Değişiklikleri görüntüle
git diff

# Commit geçmişi
git log
```

## Dev Branch İşlemleri

### Dev Branch Oluştur ve Geç
```bash
# Dev branch oluştur ve geç
git checkout -b dev

# veya mevcut dev branch'e geç
git checkout dev
```

### Dev Branch'ten Pull (Güncelleme Çek)
```bash
git pull origin dev
```

### Dev Branch'e Push (Değişiklikleri Gönder)
```bash
# Değişiklikleri ekle
git add .

# Commit yap
git commit -m "Dev: Commit mesajı buraya"

# Dev branch'e push
git push origin dev
```

### Dev Branch'ten Main'e Merge (Birleştir)
```bash
# Main branch'e geç
git checkout main

# Main branch'i güncelle
git pull origin main

# Dev branch'i main'e merge et
git merge dev

# Merge edilen değişiklikleri push et
git push origin main
```

## Genel Git Komutları

### Yeni Branch Oluştur
```bash
git checkout -b yeni-branch-adi
```

### Branch'i Sil
```bash
git branch -d branch-adi
```

### Tüm Branch'leri Görüntüle
```bash
git branch -a
```

### Remote Repository'yi Görüntüle
```bash
git remote -v
```

### Remote Repository Ekle
```bash
git remote add origin https://github.com/murathan-cotuk/belucha.git
```

### Değişiklikleri Geri Al
```bash
# Son commit'i geri al (değişiklikler korunur)
git reset --soft HEAD~1

# Son commit'i geri al (değişiklikler silinir)
git reset --hard HEAD~1
```

### Stash (Geçici Saklama)
```bash
# Değişiklikleri geçici olarak sakla
git stash

# Stash'lenmiş değişiklikleri geri getir
git stash pop

# Tüm stash'leri görüntüle
git stash list
```

## Çalışma Akışı (Workflow)

### Geliştirme Yaparken (Dev Branch)
```bash
# 1. Dev branch'e geç
git checkout dev

# 2. Son değişiklikleri çek
git pull origin dev

# 3. Değişikliklerini yap

# 4. Değişiklikleri ekle
git add .

# 5. Commit yap
git commit -m "Dev: Yeni özellik eklendi"

# 6. Push et
git push origin dev
```

### Production'a Çıkarken (Main Branch)
```bash
# 1. Dev branch'teki son değişiklikleri commit et
git checkout dev
git add .
git commit -m "Dev: Son değişiklikler"
git push origin dev

# 2. Main branch'e geç
git checkout main

# 3. Main branch'i güncelle
git pull origin main

# 4. Dev branch'i main'e merge et
git merge dev

# 5. Main branch'e push et
git push origin main
```

## Önemli Notlar

- **Main Branch**: Production için kullanılır, sadece test edilmiş kodlar buraya gönderilir
- **Dev Branch**: Geliştirme için kullanılır, yeni özellikler ve düzeltmeler burada yapılır
- Her commit'te anlamlı mesajlar yazın
- Push yapmadan önce mutlaka pull yapın
- Merge işlemlerinden önce her iki branch'i de güncelleyin
